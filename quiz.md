## Rule
**1. 진행 개요**
- 본 퀴즈는 상대 팀의 코드를 분석·이해하는 능력 + 본인 프로젝트의 이해도를 평가하기 위한 경쟁형 퀴즈이다.
- 모든 문제는 상대 팀이 작성한 코드를 기반으로 출제한다.

**2. 시간 구성 (20분 내)**
- 상대 팀 코드 분석 및 출제 시간 (10분)
- 상대 팀의 코드를 읽고 구조, 로직 등을 분석한다.
- 본인 프로젝트 문제 대응 시간 (10분)
- 분석한 코드를 기반으로 문제를 출제하고,
- 출제자는 정답(답안)을 사전에 반드시 작성해야 한다.
- 다른 팀이 이전에 출제한 문제에 대해서는 출제할 수 없다.

**채점**
- 문제를 맞힌 팀 -> 문제당 1점
- 출제한 문제를 상대 팀 중 아무도 풀지 못한 경우 -> 출제 팀에게 2점
- 부분 점수 없음 (정답 / 오답만 판정)

**문제 1**  
ultrasonic_lidar_ws의 메인 launch를 실행시키면 scan_cluster_marker가 rqt_graph에 뜨게 될텐데 이때, scan_cluster_marker 노드가 비정상적으로 종료되었을 경우 rqt_graph 상에서 /scan 토픽의 연결 상태는 어떻게 되는가?  
**답:** scan_cluster_marker 노드는 /scan 토픽의 유일한 구독자이므로, 노드가 비정상 종료되면 /scan 토픽은 퍼블리셔인 rplidar_node만 존재하고 구독자가 없는 상태로 rqt_graph에 나타난다. [rplidar_node ──▶ /scan (구독자 없음)]

**문제 2**  
ultrasonic_lidar_ws에서 하나의 장애물을 하나의 마커로 병합하려고 할 때, 여러 개의 클러스터가 하나로 병합되지 않고 남아있을 때, 1팀의 ultrasonic_lidar_ws에서 어떤걸 어떻게 수정시켜야 하는가?

**문제 3**

**문제 4**  
ultrasonic_lidar_ws/perception_stack/scan_cluster_node.py에서 def hsv_to_rgba라는 함수가 있는데 rgba에서 a가 의미하는것은?

## TEAM 4 (정가용, 정규민)

**문제 1**  
scan_cluster_marker.py에서 사용한 클러스터링 방법을 2단계에 걸쳐 설명하시오.  
**답:** 1차: 점과 점 사이 거리가 20cm 이내면 동일 물체로 간주, 노이즈 제거를 위해 점이 최소 3개 이상 모여야 유효한 물체 / 2차: 1차 클러스터 중심점 간 거리가 0.3m 이내면 하나의 덩어리로 합침

**문제 2**  
ultrasonic 토픽이 [25.0, 8.0, 15.0, 7.0, 34.0, 10.0]일 때 sound_node.py에게 기대되는 결과와 그 이유는?  
**답:** 아무 소리도 출력되지 않는다. threshold_cm이 5cm로 설정되어 있기 때문이다.

**문제 3**  
scan_cluster_nody.py에서 라이다로 인지된 정보를 바탕으로 클러스터링하는 과정에서 cluster_tolerance에 대한 파라미터가 선언되어 있는데, 이 값을 너무 작게 줄이면 어떤 문제가 발생할 수 있는지 설명하시오.  
**답:** cluster_tolerance를 너무 작게 설정하면 같은 장애물의 표면 점들 사이 간격도 끊어져, 하나의 장애물이 여러 개의 클러스터(여러 장애물)로 분리되어 인식될 수 있다.

**문제 4**  
신호등의 색깔 변화가 실제로 0.5초마다 발생하는 상황에서, speaker.launch.py의 음성 출력과 관련된 hold_sec 파라미터 값을 1.0으로 설정하면 빨간불 -> 노란불 -> 초록불로 변화하는 과정에서 어떤 문제가 발생할 수 있을까요?  
**답:** 실제 신호는 0.5초마다 바뀌는데 hold_sec를 1.0초로 늘리면 이전 색이 너무 오래 유지되어, 색이 바뀌었음에도 예전 색으로 인식될 수 있다. 그 결과 음성 출력이 늦어지거나, 중간 색(예: 노란불)이 스킵되는 문제가 발생할 수 있다.
